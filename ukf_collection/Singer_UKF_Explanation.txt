1. OVERVIEW
-----------
The Singer UKF models target acceleration as a first-order Markov process
that decays exponentially toward a baseline value. This is more realistic
than constant acceleration for maneuvering targets, as it assumes the target
cannot sustain arbitrary acceleration indefinitely.

  State vector (9 states):
    x = [x_pos, x_vel, x_acc, y_pos, y_vel, y_acc, z_pos, z_vel, z_acc]

  Measurements (3 observations):
    z = [z_x, z_y, z_z]   (position-only)

  Fixed-point format: Q24.24 (48-bit signed, 24 integer + 24 fractional bits)
  Sampling period: dt = 0.02 s (50 Hz)
  Sigma points: 2n+1 = 19 points (n=9 states)


2. MOTION MODEL (Singer's Acceleration Model)
-----------------------------------------------
Singer's model introduces exponential decay of acceleration toward a
baseline mean (a_mean). The key parameter is TAU (time constant):

    acc' = a_mean + (acc - a_mean) * exp(-dt/TAU)
    vel' = vel + (acc - a_mean) * TAU * [1 - exp(-dt/TAU)] + a_mean * dt
    pos' = pos + vel*dt + (acc - a_mean)*[dt - TAU*(1 - exp(-dt/TAU))]
                        + a_mean * 0.5 * dt^2

Parameters:
    TAU = 2.0 s       Time constant for acceleration decay
    sigma_a = 5.0 m/s^2  Acceleration standard deviation
    a_mean = 0.0 m/s^2   Baseline acceleration (configurable at runtime)

When TAU -> infinity, Singer's model reduces to constant acceleration.
When TAU -> 0, acceleration decays instantly (constant velocity).

Key property: exp(-dt/TAU) controls how much acceleration persists.
  With TAU=2.0 and dt=0.02: exp(-0.01) = 0.99 (99% retention per step)
  Over 1 second (50 steps): 0.99^50 = 0.605 (40% decay)

The exponential is computed via CORDIC (exp_cordic.vhd).


3. UKF ALGORITHM FLOW
----------------------
The algorithm follows the same predict-update structure as CA UKF, but
with Singer's prediction model and Singer-specific process noise:

  Phase 1: PREDICTION (via prediction_phase_p_3d.vhd)
    a) Cholesky decomposition of diagonal P (off-diag inputs zeroed)
    b) Sigma point generation (sigma_3d.vhd)
    c) Singer model propagation (predicti_singer3d.vhd)
       - Uses exp_cordic.vhd for exp(-dt/TAU) computation
       - Latency: ~31 clock cycles (vs 4 for CA model)
    d) Predicted mean (predicted_mean_3d.vhd)
    e) Covariance reconstruction (covariance_reconstruct_3d.vhd)
    f) Singer process noise addition (process_noise_singer_p_3d.vhd)
       - Diagonal Q with analytical Singer noise values
       - Includes saturation at MAX_VALUE = 2^31-1

  Phase 2: MEASUREMENT UPDATE (via measurement_update_3d.vhd)
    a-f) Same as CA UKF (innovation, S, Pxz, K, Joseph update)

6. ARCHITECTURAL NOTES
-----------------------
  a) THREE process noise files exist -- ONLY ONE IS ACTIVE:
       process_noise_singer_3d.vhd         -- NOT USED (dead code)
       process_noise_singer_3d_simple.vhd  -- NOT USED (L-path variant)
       process_noise_singer_p_3d.vhd       -- ACTIVE (P-matrix path)

     The top-level ukf_supreme_3d.vhd instantiates prediction_phase_p_3d,
     which uses process_noise_singer_p_3d.

  b) Diagonal-P architecture:
     Off-diagonal P elements are zeroed at the Cholesky input in
     prediction_phase_p_3d.vhd. This forces a diagonal-only covariance
     decomposition, which is always positive semi-definite. The Joseph-form
     update produces full off-diagonal terms, but they are discarded at the
     next prediction cycle.

  c) Additional (unused) modules in the source directory:
       prediction_phase_3d.vhd             -- L-factor path (not used)
       process_noise_3d.vhd                -- CA-style Q (not used)
       process_noise_rank1_3d.vhd          -- Rank-1 update variant
       qr_decomp_9x19.vhd                 -- QR decomposition (unused)
       cholesky_rank1_update.vhd           -- Square-root rank-1 update
       cholesky_rank1_downdate.vhd         -- Square-root rank-1 downdate
       state_update_joseph_3d.vhd          -- Alternative Joseph form

       ukf_supreme_3d_L_version_backup.vhd -- Backup of L-factor top-level

9. STRENGTHS AND LIMITATIONS
-----------------------------
  Strengths:
    + Physically motivated acceleration decay model
    + TAU parameter controls maneuver persistence
    + Natural damping prevents unbounded acceleration estimates
    + Analytical process noise Q derived from Singer theory
    + Suitable for maneuvering targets (drones, aircraft, missiles)

  Limitations:
    - Higher prediction latency (~31 cycles vs 4 for CA)
    - Requires CORDIC exponential hardware (exp_cordic.vhd)
    - TAU must be tuned per application (2.0s for drone, different for F1)
    - P saturation needed to prevent covariance explosion
    - Diagonal-P constraint limits theoretical accuracy
