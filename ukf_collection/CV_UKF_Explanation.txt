===============================================================================
     CONSTANT VELOCITY (CV) UNSCENTED KALMAN FILTER - VHDL IMPLEMENTATION
===============================================================================

1. OVERVIEW
-----------
The Constant Velocity UKF is the simplest of the three motion models. It
tracks a 3D target by estimating only position and velocity for each axis.
It assumes velocity remains constant between measurement updates, with no
acceleration state.

  State vector (6 states):
    x = [x_pos, x_vel, y_pos, y_vel, z_pos, z_vel]

  Measurements (3 observations):
    z = [z_x, z_y, z_z]   (position-only)

  Fixed-point format: Q24.24 (48-bit signed, 24 integer + 24 fractional bits)
  Sampling period: dt = 0.1 s (10 Hz)
  Sigma points: 2n+1 = 13 points (n=6 states)


2. MOTION MODEL (Constant Velocity)
-------------------------------------
The CV model is the simplest linear prediction:

    pos' = pos + vel * dt
    vel' = vel                (velocity is constant)

Applied identically to all 3 axes (X, Y, Z).

Key property: No acceleration state exists. Any acceleration in the real
target appears as process noise. This means the filter relies entirely on
velocity persistence and measurement corrections.


3. UKF ALGORITHM FLOW
----------------------
The algorithm follows the standard UKF predict-update cycle:

  Phase 1: PREDICTION
    a) Cholesky decomposition of 6x6 covariance P (cholsky_6.vhd)
       - Smaller matrix than CA/Singer (6x6 vs 9x9)
       - 21 unique elements (vs 45 for 9x9)
    b) Sigma point generation (sigma_3d.vhd)
       - 13 sigma points (vs 19 for CA/Singer)
    c) CV model propagation (predicti_cv3d.vhd)
       - Simple: pos += vel*dt, vel unchanged
       - Latency: ~3 clock cycles (fastest of all three models)
    d) Predicted mean (predicted_mean_3d.vhd)
    e) Covariance reconstruction (covariance_reconstruct_3d.vhd)
       - 6x6 symmetric matrix (21 unique elements)
    f) Process noise addition (process_noise_3d.vhd)
       - Diagonal Q: position and velocity variances only

  Phase 2: MEASUREMENT UPDATE
    a) Measurement mean (measurement_mean_3d.vhd)
    b) Innovation (innovation_3d.vhd)
    c) Innovation covariance (innovation_covariance_3d.vhd)
       - S = sum(W_i * dz_i * dz_i^T) + R    (3x3)
    d) Cross-covariance (cross_covariance_3d.vhd)
       - Pxz (6x3 matrix, smaller than CA/Singer's 9x3)
    e) Kalman gain (kalman_gain_3d.vhd)
       - K = Pxz * S^-1 (6x3 matrix)
    f) State update (state_update_3d.vhd)
       - Joseph-form covariance update (6x6 matrix)


4. MODULE HIERARCHY
-------------------
  ukf_supreme_3d.vhd                     [Top-level coordinator / FSM]
    |
    +-- prediction_phase_3d.vhd           [Prediction phase wrapper]
    |     +-- cholsky_6.vhd               [6x6 Cholesky decomposition]
    |     |     +-- sqrt_newton.vhd       [Newton's method square root]
    |     +-- sigma_3d.vhd               [Sigma point generation (13 pts)]
    |     +-- predicti_cv3d.vhd           [CV motion model propagation]
    |     +-- predicted_mean_3d.vhd       [Weighted mean of sigma points]
    |     +-- covariance_reconstruct_3d.vhd  [P from sigma point deviations]
    |     +-- process_noise_3d.vhd        [Add diagonal Q to P]
    |
    +-- measurement_update_3d.vhd         [Measurement update wrapper]
          +-- measurement_mean_3d.vhd     [Predicted measurement mean]
          +-- innovation_3d.vhd           [Measurement residual]
          +-- innovation_covariance_3d.vhd [Innovation covariance + R]
          +-- cross_covariance_3d.vhd     [Pxz cross-covariance]
          +-- kalman_gain_3d.vhd          [K = Pxz * S^-1]
          |     +-- matrix_inverse_3x3.vhd [3x3 matrix inverse]
          +-- state_update_3d.vhd         [Joseph-form update]


5. KEY DIFFERENCES FROM CA AND SINGER MODELS
----------------------------------------------
  +---------------------+----------+----------+---------+
  | Feature             | CV       | CA       | Singer  |
  +---------------------+----------+----------+---------+
  | States              | 6        | 9        | 9       |
  | Sigma points        | 13       | 19       | 19      |
  | P matrix elements   | 21       | 45       | 45      |
  | Cholesky size       | 6x6      | 9x9      | 9x9    |
  | Prediction latency  | ~3 cyc   | ~4 cyc   | ~31 cyc |
  | Acceleration state  | No       | Yes      | Yes     |
  | Accel decay (TAU)   | N/A      | None     | Yes     |
  | Sampling rate       | 10 Hz    | 50 Hz    | 50 Hz   |
  | Square root method  | Newton   | CORDIC   | CORDIC  |
  | Q parameters        | 2 (pos,  | 3 (pos,  | 3 (pos, |
  |                     |  vel)    |  vel,acc)|  vel,acc)|
  | Kalman gain size    | 6x3      | 9x3      | 9x3     |
  | CORDIC exp needed   | No       | No       | Yes     |
  +---------------------+----------+----------+---------+


6. KEY PARAMETERS
------------------
  Process noise Q (diagonal):
    Q_pos = 5.0 m^2            (83,886,080 in Q24.24)
    Q_vel = 0.25 (m/s)^2       (4,194,304 in Q24.24)

  Measurement noise R (diagonal):
    R_x = R_y = R_z = configured per application

  UKF weights (n=6, alpha and scaling configured per application):
    13 sigma points with appropriate mean/covariance weights


7. FILE LIST (18 modules)
--------------------------
  ukf_supreme_3d.vhd            - Top-level UKF coordinator FSM
  prediction_phase_3d.vhd       - Prediction phase wrapper
  predicti_cv3d.vhd             - CV motion model (13 sigma points)
  process_noise_3d.vhd          - Add diagonal Q to predicted P
  cholsky_6.vhd                 - 6x6 Cholesky decomposition (L*L^T = P)
  sqrt_newton.vhd               - Newton's method square root
  sigma_3d.vhd                  - Sigma point generation (13 points)
  predicted_mean_3d.vhd         - Predicted state mean from sigma points
  covariance_reconstruct_3d.vhd - P reconstruction from sigma deviations
  measurement_update_3d.vhd     - Measurement update phase wrapper
  measurement_mean_3d.vhd       - Predicted measurement mean
  innovation_3d.vhd             - Measurement residual (nu = z - z_hat)
  innovation_covariance_3d.vhd  - Innovation covariance S + R (3x3)
  cross_covariance_3d.vhd       - Pxz cross-covariance (6x3)
  kalman_gain_3d.vhd            - Kalman gain K = Pxz * inv(S) (6x3)
  matrix_inverse_3x3.vhd        - 3x3 matrix inverse (Cramer's rule)
  inverse_newsy.vhd             - Alternative matrix inverse module
  state_update_3d.vhd           - Joseph-form state/covariance update


8. STRENGTHS AND LIMITATIONS
-----------------------------
  Strengths:
    + Smallest state space (6 states) = fastest computation
    + Fewest sigma points (13) = least hardware resources
    + Smallest covariance matrix (6x6, 21 elements)
    + Simplest prediction model (~3 cycles)
    + Most numerically stable (fewer states to couple)
    + Best for targets with near-constant velocity (ships, slow UAVs)
    + No CORDIC exponential hardware needed
    + Uses Newton's method for sqrt (simpler than CORDIC)

  Limitations:
    - Cannot model acceleration explicitly
    - All target acceleration appears as process noise
    - Poor tracking of maneuvering targets
    - Higher latency response to velocity changes
    - Must use larger Q to compensate for unmodeled acceleration
    - 10 Hz sampling rate limits temporal resolution


9. WHEN TO USE EACH MODEL
---------------------------
  CV Model:
    - Slow-moving targets with minimal acceleration
    - Resource-constrained FPGA implementations
    - When acceleration information is not needed
    - Low update rate (10 Hz) applications

  CA Model:
    - Targets with sustained acceleration phases
    - When acceleration estimate is needed for control
    - Higher update rate (50 Hz) applications
    - Good balance of complexity vs. tracking ability

  Singer Model:
    - Maneuvering targets with varying acceleration
    - When acceleration is expected to decay over time
    - Drone/aircraft tracking with known maneuver characteristics
    - Best theoretical accuracy (with proper TAU tuning)
    - When extra FPGA resources and latency are acceptable
